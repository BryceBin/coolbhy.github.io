使用python自动爬取网站图片

发表于 2018-02-06 |

前言

爬虫指的是向网站发起请求，获取资源后分析并提取有用数据的程序

安装环境

python版本：3.6.4

IDE：pycharm

使用到的库

我们使用python自带的urllib对网站进行发送请求和下载图片
获取到网页资源信息后我们使用BeautifulSoup来对信息进行分析和提取。pycharm中可以在file->setting->project Interpreter中下载BeautifulSoup
得到我们想要的图片后使用os库对图片进行本地保存

准备工作

首先需要确定爬取的网站：meitu
访问网站，确定我们想要的图片(^▽^)，进入开发者工具，找到图片对应的img标签。我们发现，我们想要的图片的alt属性是非空的，而其他不想要的图片的alt属性为空，根据这个特性，我们可以从页面中将图片的网络地址筛选出来。

发送请求

当我们访问网站时，会携带着请求头，让网站识别知道我们是一个浏览器。一般网站会检测User-Agent，所以我们在程序中需要添加含有User-Agent的请求头，以便伪装成普通浏览器。User-Agent可以在开发者工具中的network选项中找到。

所以可以相应地设置我们的请求头了

返回的page就是网页的源码啦。

页面解析

根据返回的网页源码和准备工作中发现的特性，可以使用BeautifulSoup对page进行分析，直接寻找所有alt属性不为空的img标签，即可得到封面图片的描述。

现在我们得到了页面上所有封面图片的描述了，接下来，可以发现，点击封面图片后会跳转到另一个页面，里面有封面图片对应的所有图片，这才是我们最终想要的结果！(^▽^)。

所以接下来需要先得到点击封面图片后跳转到的页面的url，然后再对它进行解析。在开发者工具中发现所有封面图片对应的url都是以“http://www.win4000.com/meinv1“开头的。所以我们可以直接获取网页上所有的a标签，然后匹配其中的href属性，如果是以此字符串为开头，则存下来即可。

进入到跳转后的页面我们又怎么找到我们想要的图片的网络地址呢？仍然是在开发者工具中进行分析。可以发现，跳转页面中的大图都是存在img标签中的，并且class=”pic-large”,图片地址放在data-original属性中。

但是每页只有一张大图，需要点击下一页才能找到下一张大图。经过分析，可以发现它们的域名都是有规律的，如第一张大图是：http://www.win4000.com/meinv148873.html，那么第i张图片就是：http://www.win4000.com/meinv148873_i.html（i>=2),那么我们直接去修改对应的域名然后再去访问即可。
又发现每组图片的数量可能不一样，需要提前知道数量才方便我们进行爬取。所以在页面中分析，找到表示数量的标签”em”，然后获取它的内容就可以了。
